# Stock板块开发进度文档

## 项目背景
LightMall是一个基于Spring Boot的电商系统，目前已经实现了用户服务、商品服务和订单服务三个模块。现在需要实现stock板块的基础增删改查功能，以管理商品库存信息。

## 技术栈
- Spring Boot 2.7.15
- MyBatis Plus 3.5.3.1
- MySQL 8.0.33
- Knife4j 3.0.3 (API文档工具)

## 完成的工作

### 1. 项目结构搭建
- 创建了 `lightmall-stock` 模块
- 搭建了基本的目录结构，包括：
  - src/main/java/com/lightmall/stock/
    - config/         # 配置类
    - controller/     # 控制器
    - mapper/         # 数据访问层
    - model/          # 模型类
    - service/        # 服务层
    - StockApplication.java  # 应用启动类
  - src/main/resources/
    - mapper/         # MyBatis映射文件
    - application.yml  # 应用配置
    - init.sql        # 初始化SQL
  - pom.xml          # 项目依赖配置

### 2. 数据模型设计
- 设计了库存表结构 `t_stock`，包含以下字段：
  - id (主键，自增)
  - goods_id (商品ID，外键)
  - stock_num (库存数量)
  - lock_num (锁定数量)
  - available_num (可用数量，计算字段：stock_num - lock_num)
  - status (状态：0-不可用，1-可用)
  - create_time (创建时间)
  - update_time (更新时间)
  - version (版本号，用于乐观锁)
- 创建了模型类：
  - `Stock.java`：库存模型
  - `StockDTO.java`：库存创建请求对象
  - `StockUpdateDTO.java`：库存更新请求对象
  - `StockOperationDTO.java`：库存操作请求对象

### 3. 数据访问层实现
- 创建了 `StockMapper.java` 接口，继承 BaseMapper<Stock>，并添加了以下方法：
  - decreaseStock：扣除库存
  - increaseStock：增加库存
  - lockStock：锁定库存
  - unlockStock：解锁库存
- 创建了 `StockMapper.xml` 映射文件，实现了库存操作的SQL语句：
  - 实现了库存扣减、增加、锁定和解锁的SQL语句，使用乐观锁确保并发安全

### 4. 服务层实现
- 创建了 `StockService.java` 接口，定义了基本的CRUD方法和库存操作相关的方法
- 实现了 `StockServiceImpl.java`，实现了基本的CRUD方法和库存操作相关的方法，包括：
  - 库存锁定和解锁逻辑
  - 库存数量更新逻辑
  - 库存状态管理

### 5. 控制器层实现
- 创建了 `StockController.java`，实现了库存相关的API接口，包括：
  - 获取库存列表 (`/api/stock/list`)
  - 获取库存详情 (`/api/stock/{id}`)
  - 根据商品ID获取库存 (`/api/stock/goods/{goodsId}`)
  - 创建库存 (`/api/stock`)
  - 更新库存 (`/api/stock`)
  - 删除库存 (`/api/stock/{id}`)
  - 更新库存状态 (`/api/stock/status/{id}/{status}`)
  - 扣除库存 (`/api/stock/decrease`)
  - 增加库存 (`/api/stock/increase`)
  - 锁定库存 (`/api/stock/lock`)
  - 解锁库存 (`/api/stock/unlock`)
- 所有接口都支持JSON请求体，使用DTO对象接收参数

### 6. 配置文件配置
- 配置了 `application.yml`，包括：
  - 服务端口（9084）
  - 数据库连接信息
  - MyBatis Plus配置
- 配置了 `Knife4jConfig.java`，配置了API文档工具
- 创建了 `init.sql`，创建了库存表结构并插入了测试数据

### 7. 功能测试
- 测试了所有API接口，确保它们能够正常工作
- 测试了库存操作的并发安全性，使用乐观锁确保数据一致性
- 测试了Knife4j API文档，确保文档能够正常访问和使用

### 8. 问题解决
- 解决了编译错误：StockController.java中使用了泛型Result，但Result类不是泛型类
- 解决了运行时错误：数据库表t_stock中没有stock_num字段
- 解决了400错误：库存操作接口期望表单参数，但用户使用了JSON请求体
- 解决了500错误：MyBatis参数绑定错误，找不到参数'quantity'
