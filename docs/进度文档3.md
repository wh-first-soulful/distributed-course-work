## 遇到的问题及解决方案

### 1. 表名冲突问题
- **问题**：获取订单列表时出现 SQL 语法错误，提示 "You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near 'order'"
- **原因**：Order 模型类没有指定表名，MyBatis-Plus 默认使用类名 "order" 作为表名，而 "order" 是 MySQL 的关键字
- **解决方案**：
  - 在 Order 模型类上添加了 `@TableName("t_order")` 注解，指定正确的表名
  - 重新编译并重启服务，问题解决

### 2. 数据模型设计问题
- **问题**：订单状态与数据库表结构不一致
- **原因**：初始设计的订单状态与数据库表中定义的状态值不匹配
- **解决方案**：
  - 修改了订单状态的描述，使其与数据库表结构一致（0:待支付,1:已支付,2:已取消,3:已退款）
  - 更新了相关的业务逻辑代码

### 3. 业务逻辑完善
- **问题**：创建订单时缺少订单号生成和金额计算逻辑
- **原因**：初始实现中直接使用了传入的订单对象，没有实现必要的业务逻辑
- **解决方案**：
  - 在 OrderServiceImpl 中实现了订单号生成逻辑（时间戳+随机数）
  - 实现了订单金额计算逻辑（商品价格 × 数量）
  - 添加了订单状态的初始设置
