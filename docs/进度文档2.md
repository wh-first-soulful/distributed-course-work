## 遇到的问题及解决方案

### 1. Lombok 注解问题
- **问题**：Lombok 注解没有正确生成 getter/setter 方法，导致编译失败
- **原因**：Lombok 依赖配置不正确，注解处理器没有被正确启用
- **解决方案**：
  - 修改了 pom.xml 中的 Lombok 依赖配置，将 `optional` 标记改为 `scope provided`
  - 为所有模型类和DTO类手动添加了 getter/setter 方法，确保编译通过

### 2. BeanUtils.copyProperties 问题
- **问题**：使用 BeanUtils.copyProperties 复制属性时，属性没有被正确复制，导致创建商品失败
- **原因**：BeanUtils.copyProperties 可能无法正确处理某些属性类型或命名差异
- **解决方案**：
  - 在 Controller 中直接使用手动 setter 方法设置属性，而不是使用 BeanUtils.copyProperties
  - 确保所有属性都被正确设置，包括必填字段

### 3. 数据库字段问题
- **问题**：创建商品时出现 "Field 'goods_name' doesn't have a default value" 错误
- **原因**：数据库表中某些字段没有默认值，且传入的数据为 null
- **解决方案**：
  - 确保所有必填字段都被正确设置
  - 在创建商品时传入所有必要的字段值
